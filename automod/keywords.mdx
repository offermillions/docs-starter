The keyword filter blocks messages containing words on your list. Discord's AutoMod removes them before other members see them.

<Note>
  Your server must have **Community** enabled before using keyword commands. All commands require the **Manage Server** permission.
</Note>

## Adding a keyword

```text
,automod keywords add (word)
```

**Example** — block messages containing "badword":

```text
,automod keywords add badword
```

Keywords are stored in lowercase and matching is case-insensitive.

**Limits:**
- Maximum **60 characters** per keyword
- Maximum **1000 keywords** total

## Removing a keyword

```text
,automod keywords remove (word)
```

**Example** — stop blocking "badword":

```text
,automod keywords remove badword
```

## Viewing keywords

```text
,automod keywords list
```

## Tips

<Tip>
  You can add multi-word phrases as a single keyword, as long as the full phrase is under 60 characters.
</Tip>

<Warning>
  Discord's keyword matching is exact — a keyword of `bad` will match any message containing the string "bad", including words like "badly" or "badminton". Add keywords carefully to avoid false positives.
</Warning>

## Exempting roles or channels

If certain roles or channels should be able to use blocked words (for example, a moderation team discussing flagged content), you can exempt them from the keywords rule. See [Exemptions](/automod/exemptions).
