fault gives you three commands for controlling message-based AutoMod rules: one for spam detection, one for invite links, and one for capping the number of mentions per message. Each creates a native Discord AutoMod rule that Discord enforces at the platform level.

<Note>
  Your server must have **Community** enabled before using these commands. All commands require the **Manage Server** permission.
</Note>

## Antispam

The `antispam` rule enables Discord's built-in spam detection. Running the command again disables it.

```text
,automod antispam
```

This is a toggle — if the rule is off, running the command turns it on. If the rule is already on, running it again turns it off.

## Anti-invites

The `antiinvites` rule blocks messages containing Discord invite links (`discord.gg/*`, `discord.com/invite/*`, and `discordapp.com/invite/*`). Like antispam, it toggles on and off.

```text
,automod antiinvites
```

<Tip>
  If you have channels where invite links should be allowed (such as a staff channel), you can exempt them. See [Exemptions](/automod/exemptions).
</Tip>

## Mention limits

The `mentions` rule blocks any message that contains more than a set number of mentions. You provide the limit as a number between 2 and 50.

```text
,automod mentions (limit)
```

If you don't specify a limit, it defaults to 5. Running the command again with a different number updates the existing rule.

**Examples:**

Block messages with more than 5 mentions (default):

```text
,automod mentions
```

Block messages with more than 10 mentions:

```text
,automod mentions 10
```

Block messages with more than 2 mentions:

```text
,automod mentions 2
```

<Note>
  Running `,automod mentions` with a new limit updates the existing rule — it does not create a second rule.
</Note>

## Disabling a rule

To turn off one of these rules without toggling it, use `,automod disable`:

```text
,automod disable antispam
,automod disable antiinvites
,automod disable mentions
```

For more on disabling rules and setting up exemptions, see [Exemptions & disabling rules](/automod/exemptions).
