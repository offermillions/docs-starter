---
title: "Get started with fault: invite, setup, first commands"
description: "Learn how to add fault to your Discord server, run the moderation setup command, and use your first warn, kick, and ban commands in minutes."
---

fault is ready to use the moment you add it — no dashboard to configure before you start. This guide walks you through inviting the bot, verifying it's working, and running your first commands.

<Steps>
  <Step title="Invite fault to your server">
    Open the invite link below in your browser. Select the server you want to add fault to and click **Authorize**.

    ```text
    https://discord.com/oauth2/authorize?client_id=1496266309053186058&permissions=8&scope=bot
    ```

    <Note>
      The invite requests Administrator permissions so fault can manage roles, channels, and messages without running into permission errors. You can restrict this after setup if your server requires it.
    </Note>

    Once authorized, fault will appear in your server's member list and post a message confirming it joined.
  </Step>

  <Step title="Confirm fault is responding">
    In any channel fault can read, run the help command:

    ```text
    ,help
    ```

    fault will reply with a list of all available command categories. The default prefix is `,` (comma). You can also trigger any command by mentioning the bot instead of using the prefix:

    ```text
    @fault help
    ```
  </Step>

  <Step title="Run setup to configure moderation">
    The `,setup` command creates everything fault needs to log moderation actions:

    ```text
    ,setup
    ```

    This creates:
    - A **#mod-log** channel where all moderation actions are recorded
    - A **Muted** role for full mutes
    - An **Image Muted** role to block image and file uploads
    - A **Reaction Muted** role to block reactions

    To confirm your configuration was applied, run:

    ```text
    ,setup view
    ```

    <Tip>
      Make sure fault's role is positioned above the Muted roles in your server's role list (**Server Settings → Roles**). Discord requires a bot to outrank any role it assigns to members.
    </Tip>
  </Step>

  <Step title="Set a custom prefix (optional)">
    If the default `,` prefix conflicts with another bot, you can change it:

    ```text
    ,prefix set !
    ```

    After running that command, use `!` as your prefix going forward — for example, `!help`. You can set it to any single character or short string that works for your server.
  </Step>

  <Step title="Run your first moderation action">
    With setup complete, fault is ready to moderate. Here are a few commands to try:

    Warn a member:
    ```text
    ,warn @user spamming
    ```

    Kick a member:
    ```text
    ,kick @user breaking server rules
    ```

    Ban a member:
    ```text
    ,ban @user repeated violations
    ```

    Every action is logged automatically to the mod-log channel and assigned a case number you can reference later.

    <Note>
      In command syntax throughout these docs, parameters in `()` are required and parameters in `[]` are optional.
    </Note>
  </Step>
</Steps>
