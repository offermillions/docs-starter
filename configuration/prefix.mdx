---
title: "Custom prefix: configure your server's command prefix"
description: "Set a custom command prefix for your server. The default prefix is a comma. The bot also responds to @mentions regardless of prefix."
---

fault uses `,` as its default command prefix. You can change this to any string up to 10 characters to avoid conflicts with other bots or to match your server's style.

## View the current prefix

```text
,prefix
```

Displays the prefix currently set for your server.

## Set a custom prefix

```text
,prefix set (prefix)
```

Requires the **Manage Server** permission.

```text
,prefix set !
,prefix set f!
,prefix set fault.
```

Prefix must be 10 characters or fewer.

## Reset to the default prefix

```text
,prefix reset
```

Returns the prefix to `,`.

## Using @mention as a prefix

You can always use a direct @mention in place of the prefix, regardless of what the prefix is set to. This is useful if you forget the current prefix.

```text
@fault help
@fault prefix
```
