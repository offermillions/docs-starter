Every time you run a moderation command, fault creates a numbered case. Cases record who did what, to whom, and why.

## Viewing cases

### View a specific case

```text
,case 42
```

Displays the full case record for the given ID.

### View all server cases

```text
,cases
```

Lists every case on the server, paginated in groups of five, sorted newest first.

### Filter by member

```text
,cases @user
```

Shows only cases for the specified member.

## Updating a case reason

If you forgot to include a reason when running a command, or want to clarify it later:

```text
,reason 42 actually raiding — confirmed with logs
```

Updates the reason on case `#42`. The mod-log post is not retroactively edited.

<Note>
Updating a reason requires the **Manage Messages** permission.
</Note>

## Warning commands

Warnings are standard cases with the action type `warn`.

### View warnings for a member

```text
,warnings @user
```

Filters the case list to show only warnings for that member.

### View all server warnings

```text
,warnings
```

Shows all warning cases across the entire server.

### Clear all warnings for a member

```text
,clearwarns @user
```

Permanently deletes all warning cases for that member. Does not affect other case types.

<Warning>
Clearing warnings is irreversible. The deleted cases cannot be recovered.
</Warning>

## Mod-log channel

To have cases posted automatically, configure a mod-log channel:

```text
,setup modlog #mod-log
```

If no mod-log is configured, cases are still created and retrievable via `,case` and `,cases` — they just won't post to a channel. See [Moderation overview](/moderation/overview) for setup instructions.
